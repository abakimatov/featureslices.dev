## Structure

Divide all your components to two groups:
- `ui` â€” pure components aimed to be your UI library
- `features` â€” components and code, aimed to solve business-cases

### UI

UI components should be **reusable**. You can use [AtomicDesign](http://atomicdesign.bradfrost.com) to structure ui components.

**Important: No pages in UI!**

Example:

```
src/
  ui/
    atoms/
      button/
        index.ts
        stories.ts
        tests.ts
        readme.md
      input/
      label/
      badge/
    molecules/
    organisms/
    templates/
```
